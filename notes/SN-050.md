**Sections:** §8.6, §8.9
**Issue:** §8 specified that links and tags inside "fenced code blocks" and "inline code spans" are excluded from `file.links`/`file.tags`, but did not mention: (1) indented code blocks (4+ spaces per CommonMark §4.4), which should also be excluded, and (2) backslash escaping of wikilinks (`\[[` preventing wikilink parsing), which is standard behavior in Obsidian and CommonMark-adjacent tools.
**Resolution:** §8 updated. Link exclusion list now includes indented code blocks. Added escaping rule: `\[[` prevents wikilink parsing (backslash consumed, brackets treated as literal text). Tag extraction also updated to exclude indented code blocks.
