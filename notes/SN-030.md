**Sections:** §5.7
**Gap:** No conformance test explicitly verifies that parent types are resolved correctly when the parent definition is encountered *after* a child in the filesystem scan (or in a subdirectory). §5.7 requires a dependency graph and load in parent-before-child order, but tests only cover missing parent and circular inheritance.
**Assumption:** Implementations must load types in dependency order regardless of filename or scan order. Add a test where `child` extends `parent` and the parent file appears later or in a subdirectory; validate that the child inherits fields and no `missing_parent_type` is emitted.
**Resolution:** Added Level 1 test in `tests/level-1/types-basic.yaml` that places the parent type after the child and expects inherited required field validation to fail.
