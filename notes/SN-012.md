**Sections:** §11.18, Appendix C.4
**Ambiguity:** The spec mentions `expression_depth_exceeded` for "chained asFile() calls exceed 10-hop limit" but doesn't define a general maximum depth for nested expressions. The test uses a deeply nested `if()` chain (11 levels) to trigger this error. The exact depth limit is implementation-defined — the test assumes a reasonable implementation will reject extremely deep nesting.
**Assumption:** Implementations should have a configurable or default maximum expression depth (e.g., 10-20 levels) and emit `expression_depth_exceeded` when exceeded. The test uses 11 nested `if()` calls.
**Resolution:** §11.18.1 "General Expression Nesting Limit" added to 11-expressions.md defining a 64-level default limit (configurable 32–256), distinct from the 10-hop `asFile()` limit. Appendix C.4 `expression_depth_exceeded` updated to cover both limits. The depth-limit test updated from 11 to 65 nested `if()` calls. The previously contradictory expression-error-hardening.yaml test (11 levels, expects success) is now consistent.
