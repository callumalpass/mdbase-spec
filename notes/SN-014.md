**Sections:** §8.6, §8.8
**Ambiguity:** §8.6 says `file.links` returns non-embed links and `file.embeds` returns only embeds. But §8.8 says `file.backlinks` returns "files that link TO the current file." It's not explicit whether embeds (`![[target]]`) count as "linking to" a file for backlink purposes. Conceptually, an embed is a reference and should produce a backlink.
**Assumption:** Embeds create backlinks. `file.backlinks` includes files that reference the target via any link or embed, from both frontmatter and body. This matches user expectations — if a file embeds another, the embedded file should show a backlink.
**Resolution:** §8.8 now states backlinks include embeds. Backlinks tests already cover embeds.
