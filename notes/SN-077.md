**Sections:** Various
**Severity:** Medium (test completeness)
**Issue:** The following MUST requirements have no dedicated conformance test:

1. **§12.8 — MUST preserve line ending style (LF vs CRLF):** No test verifies that an update operation preserves CRLF line endings through a read-modify-write cycle. The `line_endings` setup extension (SN-022) exists but no test uses it for this specific assertion.

2. **§12.11 — MUST NOT require locking for read operations:** No test verifies that concurrent reads succeed without locks. This may be inherently untestable in the current test format.

3. **§13.3.3 — MUST detect stale cache entries:** Cache staleness detection tests are limited. No test verifies detection of all four staleness conditions (file modified after cache, file deleted but in cache, new file not in cache, config changed).

4. **§15.4 — MUST debounce filesystem events:** Watch mode tests exist but debounce coalescing (multiple rapid changes → single event) is not tested.

5. **§7.2 `unique` field constraint — MUST report `duplicate_value`:** Cross-file uniqueness checking for arbitrary fields (not just `id_field`) has no dedicated test. Only `duplicate_id` (for the id_field) is tested.

6. **§5.4 — Circular inheritance MUST be detected and rejected:** No test exercises circular type inheritance (e.g., `A extends B extends A`).

7. **§12.7 — Batch dry-run MUST validate without modifying files:** Batch dry-run is referenced in tests but no test specifically verifies that no files are modified after a dry-run.

**Proposed resolution:** Add targeted tests for items 1, 5, 6, and 7 (feasible in the current test format). Items 2, 3, and 4 may require test format extensions or be documented as requiring manual verification.
**Resolution:** Line ending preservation, unique constraints, and circular inheritance already have coverage. Added a Level 6 batch dry-run test. Remaining lock/read behavior and some cache/watch nuances are documented as out-of-scope for automated conformance.
