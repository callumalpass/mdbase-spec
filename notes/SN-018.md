**Sections:** §12.7, §14.3
**Ambiguity:** The test format in §14.3 doesn't define `batch_update` or `batch_delete` operations, nor a `batch_result` assertion format. The spec defines batch operations with MUST requirements but the conformance test format only shows single-file operations.
**Assumption:** Tests use `operation: batch_update` with `where` and `fields` inputs, and `expect.batch_result` with `total`, `succeeded`, `failed` fields. `dry_run: true` in input enables dry-run mode. Test runners must implement batch operations for these tests.
**Resolution:** §14.3.1 "Extended Test Format" added to 14-conformance.md, documenting `batch_update`, `batch_delete` as extended operations, `dry_run` as an extended input field, and `batch_result` as an extended assertion field.
