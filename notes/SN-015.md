**Sections:** §15, §14.3
**Ambiguity:** The test format in §14.3 doesn't define how to express watch mode scenarios. Watch tests need to simulate filesystem events (creates, modifies, deletes, renames) that happen externally and then assert which events the watcher emits. The tests introduce `operation: watch` with `simulate` blocks describing external filesystem changes and `expect.events` describing the expected event sequence.
**Assumption:** Test runners will interpret `operation: watch` as starting a watcher, performing the simulated filesystem changes, waiting for debounce, and then comparing emitted events against `expect.events`. Additional assertions like `expect.max_event_count`, `expect.events_ordered`, `expect.events_contain`, and `expect.listener_query` extend the basic event-matching model.
**Resolution:** §14.3.1 "Extended Test Format" added to 14-conformance.md, documenting `watch` as an extended operation.
