**Sections:** §7.11, Appendix C.1
**Ambiguity:** The `list_item_invalid` error code is defined in Appendix C but no test in Level 1 currently triggers it distinctly from `type_mismatch`. When a list item fails validation, it's unclear whether implementations should emit `list_item_invalid` (item-level) or the underlying error code (e.g., `type_mismatch` for the item). The appendix implies `list_item_invalid` wraps item-specific failures.
**Assumption:** `list_item_invalid` is emitted when a list item fails its `items` type validation. The underlying cause (type_mismatch, constraint_violation, etc.) may be included as a nested detail. Will add a test in Level 2.
**Resolution:** Appendix C clarifies `list_item_invalid` as an item-level failure. Level 1 and Level 2 tests already exercise `list_item_invalid` (spec-coverage-gaps.yaml, matching-eval.yaml).
