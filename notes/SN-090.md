**Sections:** §2.2, §12.2
**Issue:** Tests assert that direct reads of excluded subfolder files return `file_not_found`, but §12.2 says read loads from filesystem without clarifying exclusion behavior. The spec is silent on whether excluded files should be unreadable by explicit path.
**Proposed resolution:** Clarify that `settings.include_subfolders: false` and `settings.exclude` apply to reads as well as queries, or explicitly state that read bypasses scanning exclusions.
**Resolution:** §12.2 now states reads respect scanning exclusions and return `file_not_found` for excluded paths.
