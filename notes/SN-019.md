**Sections:** §3.4, §14.3
**Ambiguity:** The spec says implementations MUST NOT write the empty-value null form (`field:` with no value). To test this, we need to assert the serialized output doesn't contain bare nulls. The standard assertion types in §14.3 don't cover serialization format checking.
**Assumption:** Tests use `expect.frontmatter_not_bare_null: [field1, field2]` to assert that named fields are not written in the bare `field:` form. Test runners must inspect the serialized YAML output, not just the parsed values.
**Resolution:** §14.3.1 "Extended Test Format" added to 14-conformance.md, documenting `frontmatter_not_bare_null` as an extended assertion field.
