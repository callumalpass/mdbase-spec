**Sections:** Appendix C.3
**Ambiguity:** Both `path_traversal` and `invalid_path` can apply to `../../../etc/passwd`. The spec defines `path_traversal` specifically for link resolution escaping collection root, and `invalid_path` for malformed paths. For create/rename operations with `..` paths, the test currently expects `invalid_path`. A link-resolution test in Level 4 should trigger `path_traversal`.
**Assumption:** `invalid_path` is used when the operation input path is malformed or escapes root. `path_traversal` is used specifically during link resolution (Level 4).
**Resolution:** Appendix C.3 updated. `invalid_path` narrowed to "Path is malformed (e.g., null bytes, control characters)." `path_traversal` broadened to "Path resolution would escape collection root. Applies to any path resolution — link resolution, operation input paths, computed paths."
