**Sections:** §11.7, §7.8
**Issue:** `now()` and `today()` use the implementation's local timezone unless otherwise configured, but there is no configuration field or normative default. This makes identical data produce different query results across machines.
**Proposed resolution:** Add a `settings.timezone` option (default `UTC`) or explicitly require UTC for these functions unless configured.
**Resolution:** Added optional `settings.timezone` (default local timezone) and updated §11.7 to use it when set.
