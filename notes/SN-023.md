**Sections:** §2.8, §14.3
**Ambiguity:** To test nested collection detection, the test setup must create a subdirectory containing its own `mdbase.yaml`. The standard `setup.files` mapping allows arbitrary file paths, but it's not explicit whether test runners should interpret `sub-project/mdbase.yaml` as creating a nested collection marker.
**Assumption:** Test runners create the file at the specified path within the test collection. The implementation under test must detect the nested `mdbase.yaml` and exclude that subtree from scanning.
**Resolution:** §14.3.1 "Extended Test Format" added to 14-conformance.md, documenting nested collection support as an extended setup capability.
