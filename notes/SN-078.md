**Sections:** Various
**Severity:** Low to Medium
**Issue:** The following edge cases are not addressed by the spec or tested:

1. **Computed field referencing a `default` field:** §5.12 says computed fields are evaluated at read time. If a computed field's expression references a field with a `default`, the effective value (with default applied) should be used. This is implied by the evaluation order (§5.13: "Non-computed fields are resolved first") but not explicit.

2. **Generated field depending on another generated field:** If field A has `generated: ulid` and field B has `generated: {from: A, transform: slugify}`, the order of generation matters. The spec doesn't define generation order for interdependent generated fields.

3. **`path_pattern` variable referencing a computed field:** Computed fields are not persisted and are evaluated at read time. If `path_pattern: "{computed_field}.md"`, should the computed value be available during create? The spec is silent.

4. **Empty `values` list in enum:** What happens with `type: enum, values: []`? No value can ever be valid. Should type loading reject this?

5. **Update that changes match-relevant fields (reclassification):** If a file matched type A via `where: {status: {eq: "open"}}` and the user updates `status` to `"done"`, the file may no longer match type A. The spec doesn't address whether this should warn, error, or silently allow reclassification.

6. **Delete of a type definition file while files reference it:** If `_types/task.md` is deleted but `tasks/*.md` files still have `type: task`, those files become orphaned. The spec doesn't define whether this should be prevented, warned about, or allowed.

7. **Concurrent init operations:** Two processes running `mdbase init` in the same directory simultaneously. No atomicity guarantee is specified for init.

8. **Type with `path_pattern` containing directory that doesn't exist:** If `path_pattern: "archive/{year}/{slug}.md"` and `archive/2024/` doesn't exist, should create auto-create the directory? The spec says "write atomically (temp file + rename)" but doesn't mention directory creation.

9. **Self-referential link field:** A file with `parent: "[[self]]"` where `[[self]]` resolves to the file itself. Not prohibited but may cause infinite loops in traversal if `asFile()` is used recursively.

10. **`where` match rule with expression that errors:** If a `where` condition like `tags.contains("x")` encounters a file where `tags` is an integer (not a list), should the match silently fail or produce a warning? SN-008 says silent non-match for missing/null, but type mismatches during match evaluation are unaddressed.

**Proposed resolution:** Items 2, 4, 5, and 8 are the most likely to cause real implementation divergence and should be addressed in the spec. The rest can be noted as implementation guidance in §14.6.
**Resolution:** Added clarifications for defaults in computed fields, generated-field dependency order, computed fields in `path_pattern`, empty enum values, update reclassification, type removal behavior, init concurrency, directory creation for `path_pattern`, self-referential links, and match-rule evaluation errors. Added tests for computed defaults, generated dependencies, computed-in-path_pattern rejection, empty enum values, and directory creation.
