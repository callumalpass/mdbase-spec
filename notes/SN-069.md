**Sections:** §4.7
**Ambiguity:** The spec requires literal treatment of `${...}` when unsupported, but does not define whether partial expansion is allowed when only some variables are defined or when `${VAR:-default}` is not supported. It also does not specify whether substitution should occur in nested keys, lists, or only strings.
**Assumption:** Substitution applies to all string scalars in the config tree. If any placeholder is unsupported, the entire string remains literal (no partial expansion).
**Proposed resolution:** Define substitution as a pure string-to-string transform applied to every string scalar; unsupported syntax leaves the full string unchanged.
**Resolution:** §4.7 now defines substitution as a string-to-string transform over all string scalars with no partial expansion.
