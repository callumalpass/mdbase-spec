**Sections:** §12.5
**Issue:** references.yaml Group 1 ("frontmatter link updates on rename") and Group 4 ("ID-based link stability on rename") both configured `id_field: "id"`. Group 1 expected `[[task-b]]` to be rewritten to `[[task-beta]]` after renaming task-b.md → task-beta.md. Group 4 expected `[[task-b]]` to NOT be rewritten after renaming task-b.md → task-b-renamed.md. In both cases the `id_field` value ("task-b") was unchanged. Per §12.5: "If a simple-name link resolves via `id_field` and the target file's `id_field` value did not change, implementations SHOULD NOT rewrite the link." Group 1 contradicted this by expecting a rewrite.
**Resolution:** Removed `id_field: "id"` from Group 1's config. Group 1 now tests filename-based link resolution (no id_field → `[[task-b]]` resolves by filename → must be rewritten when filename changes). Group 4 retains `id_field` and correctly tests ID-based stability. No contradiction.
