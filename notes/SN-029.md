**Sections:** §2.9, §8.4
**Ambiguity:** §2.9 says "Implementations MUST resolve links to non-markdown files by path only (no `id_field` lookup)". But §8.4 step 4 (simple name resolution) searches by id_field then by filename. For a simple-name wikilink like `[[diagram]]`, does the filename match pass consider non-markdown files like `diagram.png`? §2.9 implies NO — non-markdown files are not records and don't participate in the simple-name resolution algorithm. They only resolve when explicitly referenced by path.
**Resolution:** §8.4 step 4 now explicitly scopes filename matching to markdown records, and includes a note that non-markdown files are not candidates for simple-name matching.
