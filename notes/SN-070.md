**Sections:** §9.4–§9.10
**Ambiguity:** The spec includes SHOULDs for explicit validation commands and skip-validation logging, but the conformance test format has no way to assert CLI/API command availability or logging side effects.
**Assumption:** These requirements are informational and should not affect automated conformance; they should be marked non-normative or moved to an implementation guidance section.
**Proposed resolution:** Either add test-runner hooks for `validate` command options and log capture, or downgrade these points to non-normative guidance.
**Resolution:** §14.4 now notes these items are out of scope for automated conformance; implementations should document compliance.
