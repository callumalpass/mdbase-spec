**Sections:** §3.2, §3.5, §14.3
**Ambiguity:** The test format in §14.3 doesn't define how to express files with specific encodings (non-UTF-8) or line ending styles (CRLF vs LF). These are byte-level properties that can't be represented in standard YAML string values.
**Assumption:** Tests use extended `setup.files` syntax:
- `encoding: "latin-1"` to indicate the file bytes should be interpreted as Latin-1 (not UTF-8)
- `line_endings: "LF"` or `line_endings: "CRLF"` to specify line ending style
- `content: "..."` as the raw content string when encoding/line_endings are specified
Test runners must create the actual files with these byte-level properties. The `expect.line_endings` assertion checks that the output file preserves the original line ending style.
**Resolution:** §14.3.1 "Extended Test Format" added to 14-conformance.md, documenting `encoding` and `line_endings` as extended setup fields.
