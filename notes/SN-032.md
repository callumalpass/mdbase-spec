**Sections:** §5.8, §12.12
**Gap:** `init` tests only assert that `_types/meta.md` exists and has `name: meta`. The required meta type schema (e.g., `match.path_glob`, `strict: false`, `fields` definitions) and the rule to adjust `match.path_glob` for a custom `settings.types_folder` are not validated.
**Assumption:** The meta type should match the documented schema and update its `match.path_glob` to the configured types folder. Add tests that read `_types/meta.md` (or custom folder) and assert key fields and `match.path_glob` correctness.
**Resolution:** Added Level 1 tests in `tests/level-1/init.yaml` to assert required meta type fields and custom types folder path_glob handling.
