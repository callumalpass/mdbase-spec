**Sections:** ¬ß7.11, ¬ß7.16
**Ambiguity:** When a list has `items: { type: string }` and contains integers like `[1, 2, 3]`, ¬ß7.16 says string accepts "any scalar (converted via toString)". So integer list items should be coerced to strings and the list should be valid. The test assumes coercion applies recursively to list items.
**Assumption:** Type coercion (¬ß7.16) applies to list items just as it applies to top-level fields.
**Tests added:** spec-coverage-gaps.yaml Group 2 explicitly tests integer‚Üístring, boolean‚Üístring, string‚Üíinteger, float‚Üíinteger coercion for list items.
**Resolution:** ¬ß7.11 now states that coercion rules apply to list elements. Existing tests cover the behavior.
