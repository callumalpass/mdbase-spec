**Sections:** §3.2, §9.1
**Issue:** validation.yaml "frontmatter YAML structure" group tested non-mapping frontmatter (list/scalar) and expected `error: { code: invalid_frontmatter }`, but the test config did not specify `default_validation`. The default is `"warn"` per §9.1. §3.2 explicitly states that non-mapping frontmatter follows the validation level: at `"warn"`, it should be treated as empty frontmatter with a warning, not an error. This contradicted conformance-edge-cases.yaml which correctly tests all three levels.
**Resolution:** Added `default_validation: "error"` to the validation.yaml test group config. The tests now explicitly use error-level validation, making the `error` expectation correct and consistent with §3.2 and conformance-edge-cases.yaml.
