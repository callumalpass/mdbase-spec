**Sections:** §12.7
**Issue:** Two L1 tests contradict on what the `details` array should contain after a partial batch failure. spec-coverage-gaps.yaml "batch with I/O failure continues processing remaining files" expected `details` with only the failed entry (1 item). batch-result-details.yaml "partial failure includes both success and failed entries in details" expected `details` with all entries (3 items: 2 success + 1 failed).
**Resolution:** Updated spec-coverage-gaps.yaml to include all entries in `details` (success and failed), matching batch-result-details.yaml which is the more explicit and intentional test of this behavior.
