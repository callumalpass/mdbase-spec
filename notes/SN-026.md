**Sections:** §4.8, §7.9
**Ambiguity:** §4.8 says implementations MUST support ECMAScript regex and reject unsupported features with a clear error. But it doesn't specify *when* invalid regex patterns should be rejected — at type definition load time, or at validation time when the pattern is first applied. Early rejection (load time) is better for user experience, but the spec doesn't explicitly mandate it.
**Assumption:** Invalid regex patterns in type definition `pattern` constraints are rejected at type load time with `invalid_type_definition`. This is the stricter interpretation and catches errors earlier.
**Resolution:** §4.8 now requires invalid regex patterns to be rejected at type load time with `invalid_type_definition`. Tests already enforce this.
