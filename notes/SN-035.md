**Sections:** §11.6
**Ambiguity:** The spec examples always pass an initial accumulator value to `reduce()`, but it never states whether the initial value is required or what happens when it is omitted (especially on empty lists). Tests only cover the initial-value form.
**Assumption:** Specify whether `reduce()` requires an initial value and define behavior for empty lists and single-element lists when omitted. Add tests once semantics are decided.
**Resolution:** Clarified §11.6 to require `init` and added a wrong-argument-count test in `tests/level-3/expression-error-hardening.yaml`.
