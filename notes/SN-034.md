title: "Non-scalar `order_by` rule conflicts with allowed rejection"

**Sections:** §10.3
**Ambiguity:** §10.3 says implementations MUST sort non-scalar fields deterministically (lists by length, objects by key count), but also says implementations MAY reject non-scalar sort fields with a warning. These requirements conflict, and current tests enforce sorting, leaving no conforming path for "reject with warning."
**Assumption:** Clarify the spec to either (a) require deterministic sorting (remove the reject-with-warning allowance), or (b) allow both behaviors and update tests to accept a warning+rejection outcome.
**Resolution:** Clarified §10.3 to require deterministic sorting and allow a warning (not rejection).
