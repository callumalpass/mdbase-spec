**Sections:** §12.5, §12.6, Appendix C.3
**Ambiguity:** The spec says rename with ref updates is not atomic across files and implementations should "report partial failures clearly." But the error code `rename_ref_update_failed` is defined as a single error. It's unclear whether this error is returned alongside a successful rename result, or if the rename operation itself is considered to have failed. The distinction matters: the renamed file exists at the new path, but some references weren't updated.
**Assumption:** The rename operation returns `rename_ref_update_failed` as an error alongside the successful rename result (`from`/`to`). The error includes details about which files failed. The renamed file is at the new path. Test runners should verify both the error and the successful rename.
**Resolution:** §12.5 updated with a "Partial Reference Update Failure" subsection defining the response format: successful rename (`from`/`to`) + `ref_update_errors` array with per-file failure details + `rename_ref_update_failed` error code.
