**Sections:** §7.15, §5.6
**Gap:** The rule "If a generated field depends on `file.*`, that field MUST NOT be referenced by `path_pattern`" is not tested. Current tests cover path_pattern with generated fields, but not the prohibited `file.*` dependency case.
**Assumption:** Type definitions that reference a `file.*`-derived generated field in `path_pattern` should be rejected at type load time (likely `invalid_type_definition`). Add a test that defines a generated field `slug: { from: file.name }` and `path_pattern: "{slug}.md"` and expects a type-definition error.
**Resolution:** Added Level 1 test in `tests/level-1/error-code-hardening.yaml` and clarified §7.15/§5.6 to require `invalid_type_definition`.
