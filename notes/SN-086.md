**Sections:** §4.8
**Issue:** If env-var expansion is supported, §4.8 requires "Only expand variables explicitly referenced" and "Never log expanded values." There is no test coverage for these constraints.
**Proposed resolution:** Add tests that verify: (1) no implicit expansion of unrelated vars, (2) expanded values are not surfaced in error messages or logs (may require test-runner support).
**Resolution:** §14.4 now documents these as out-of-scope for automated conformance; implementations should document compliance.
