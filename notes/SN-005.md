**Sections:** §12.6, §14.3
**Ambiguity:** The test format doesn't define how to assert file state after a failed operation (e.g., verifying atomicity -- that a failed update didn't corrupt the file). Tests use a `verify_after` block.
**Assumption:** Test runners will execute `verify_after.operation` after the primary test and check its `expect` block.
**Resolution:** §14.3.1 "Extended Test Format" added to 14-conformance.md, documenting `verify_after` as an extended assertion field.
