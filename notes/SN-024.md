**Sections:** §2.9, §14.3
**Ambiguity:** Tests for non-markdown file handling need to create binary files (images, PDFs) in the test collection. The test format uses string content for `setup.files` entries, but binary files can't be represented as YAML strings.
**Assumption:** Tests use placeholder string content (e.g., `"binary content"`) for non-markdown files. Test runners should create these files regardless of content — the important thing is that they exist on the filesystem. The tests verify that these files are NOT treated as records, not that their content is valid.
**Resolution:** This is a test runner implementation detail — no spec change needed. §14.3.1 documents the general setup file conventions.
