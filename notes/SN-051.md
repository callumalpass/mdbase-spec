**Sections:** §10.3
**Issue:** §10.3 defined sort semantics for strings, numbers, dates, enums, and nulls, but did not specify behavior when `order_by` references a list or object field. Without guidance, implementations would handle this differently (error, stringify, sort by length, etc.).
**Resolution:** §10.3 updated with non-scalar field guidance: sort lists by length, objects by key count. Implementations MAY reject non-scalar sort fields with a warning.
