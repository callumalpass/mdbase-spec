**Sections:** §5.8, §4.4
**Issue:** §5.8 says the meta type must adjust `match.path_glob` when `settings.types_folder` is customized, but there is no test verifying this behavior.
**Proposed resolution:** Add a Level 1 test with a custom `types_folder` that verifies the meta type matches files in the custom folder, not `_types/`.
**Resolution:** Added tests in `tests/level-1/init.yaml` for custom types folder `path_glob`.
