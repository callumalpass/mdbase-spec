**Sections:** §7.16, §11
**Issue:** §7.16 defines frontmatter coercion rules; §11 defines expression evaluation behavior. The spec does not state whether expression evaluation uses the same coercion rules as validation or a separate type system.
**Proposed resolution:** Add a cross-reference clarifying whether expression evaluation uses validation-style coercions (e.g., numeric strings) or treats values strictly by runtime type.
**Resolution:** §10.5 now clarifies that bare properties use effective/coerced values, while `note.`/`file.properties` expose raw values.
