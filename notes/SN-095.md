**Sections:** §13.3.1, §13.6, §13.4, §14.1, §14.6
**Issue:** Several normative statements use lowercase modal verbs instead of RFC 2119 uppercase keywords, inconsistent with §1's declaration.

| Location | Current | Suggested |
|----------|---------|-----------|
| §13.3.1 | `"No information should exist only in the cache."` | `MUST NOT` — central cache correctness guarantee |
| §13.6 | `"Cache entries should be invalidated when:"` | `MUST` or `SHOULD` |
| §13.4 (Gitignore note) | `"Cache files are machine-specific and should not be version controlled."` | `SHOULD NOT`, or move to a non-normative aside |
| §14.1 | `"Implementations may claim conformance at different levels."` | Rephrase as a factual statement, or `MAY` if permission is intended |
| §14.6 | Three lowercase `"should"` instances in Edge Cases and Error Messages | `SHOULD`, or clearly mark the section non-normative |

The §13.3.1 case is the most consequential: the prohibition on cache-only information is a core correctness requirement but currently reads like a suggestion.
