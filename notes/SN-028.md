**Sections:** §7.15, §14.3
**Ambiguity:** Testing `generated: now_on_write` requires asserting that a field was modified without knowing the exact value (since it's the current timestamp). The existing assertion types (`frontmatter`, `frontmatter_written`, `frontmatter_not_written`) check exact values or presence/absence.
**Assumption:** Tests use `expect.frontmatter_changed: [field1, field2]` to assert that the listed fields have different values after the operation compared to before, without specifying the new value.
**Resolution:** §14.3.1 "Extended Test Format" added to 14-conformance.md, documenting `frontmatter_changed` as an extended assertion field.
