**Sections:** §5.6
**Severity:** Low (editorial)
**Issue:** §5.6 contains two introductory sentences for `path_pattern` that contradict each other:
- Line 281: "The optional `path_pattern` defines expected **relative paths** for a type, not just filenames."
- Line 288: "The optional `path_pattern` defines expected filename structure:"

The first correctly describes the feature (relative paths including subdirectories). The second is a leftover from before SN-064 renamed `filename_pattern` to `path_pattern` and still says "filename structure."
**Proposed resolution:** Delete the second paragraph (line 288) entirely. Its content is redundant with lines 281–282 and the code example that follows.
**Resolution:** Removed the duplicate `path_pattern` introductory paragraph in §5.6.
