**Sections:** §7.5, Appendix C.1
**Ambiguity:** §7.5 says "IEEE 754 special values (NaN, Infinity) are allowed unless explicitly constrained." For Infinity, it's clear: `.inf` exceeds any finite `max`, `-.inf` falls below any finite `min`. For NaN, the behavior is unclear: NaN comparisons always return false in IEEE 754, so NaN is neither above max nor below min. Should implementations reject NaN when min/max are set? The test assumes yes — NaN with min/max constraints produces `constraint_violation` because NaN cannot satisfy the constraint (it's not a valid number in the range).
**Assumption:** NaN fails any min/max constraint with `constraint_violation`. Implementations should treat NaN as "not satisfying" any numeric range.
**Resolution:** §7.5 updated with an explicit validation bullet: "When `min` or `max` is defined, `NaN` MUST produce `constraint_violation` because `NaN` is not orderable — all comparisons with `NaN` return false per IEEE 754. Infinity values are orderable and compared normally."
