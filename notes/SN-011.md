**Sections:** Appendix C.4, Appendix C.5
**Ambiguity:** When a formula (defined in Query+ `formulas` clause) encounters a runtime error like division by zero, should it produce `formula_evaluation_error` (C.5) or `type_error` (C.4)? The spec defines both — `type_error` for "type error in expression" and `formula_evaluation_error` for "runtime error in formula". Division by zero is listed under §11.18 as returning null with `type_error`, but in the formula context it could be `formula_evaluation_error`.
**Assumption:** When a formula expression encounters a runtime error, it produces `formula_evaluation_error` (the formula-specific code). The expression-level `type_error` is used when the error occurs in a `where` filter expression or computed field, not in a formula. The test for `formula_evaluation_error` uses division by zero in a formula.
**Resolution:** §10.7 now specifies that formula runtime errors yield `formula_evaluation_error` (Appendix C.5). Formula tests already enforce this.
