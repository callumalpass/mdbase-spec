**Sections:** §2.3
**Gap:** The spec allows subdirectories inside the types folder and states all `.md` files there are scanned. There is no test that a type definition in `_types/subdir/` is loaded and that the file is excluded from record scans.
**Assumption:** Any `.md` file under the types folder hierarchy (including nested directories) is a type definition, not a record. Add a test with `_types/subdir/widget.md` and ensure type loading succeeds and the file is not queryable as a record.
**Resolution:** Added Level 1 tests in `tests/level-1/collection-layout.yaml` covering type definitions under `_types/subdir/`.
