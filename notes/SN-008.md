**Sections:** §6.4
**Ambiguity:** The spec defines `fields_present` as requiring the key to exist AND be non-null. But for `where` operators like `contains`, `startsWith`, `gte`, etc., the spec doesn't explicitly state what happens when the field is missing or null. Should it be a silent non-match, or should it produce an error?
**Assumption:** All `where` operators silently fail to match when the field is missing or null. This is consistent with `fields_present` behavior and with the idea that match rules should be "safe" — a missing field simply means the condition is not satisfied.
**Resolution:** §6.4 now explicitly states missing/null (and evaluation errors) yield a silent non-match. Matching-fields tests cover missing/null and wrong-type scenarios.
