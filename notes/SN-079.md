**Sections:** §11.8
**Issue:** The duration unit table uses `m` for minutes and `M` for months, but the spec never states whether unit parsing is case-sensitive. A case-insensitive implementation could treat `m` and `M` as the same unit, producing incorrect results.
**Proposed resolution:** Add a sentence: "Duration units are case-sensitive; `M` is months and `m` is minutes."
**Resolution:** Added case-sensitivity sentence in §11.8.
