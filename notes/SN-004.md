**Sections:** §12.11
**Ambiguity:** The spec describes mtime-based conflict detection but the test format in §14.3 doesn't define how to express concurrency scenarios. The tests use a `simulate` block with `external_modify` and `external_create` to describe side effects that occur between the read and write phases of an operation.
**Assumption:** Test runners will interpret `simulate.external_modify` and `simulate.external_create` as injecting filesystem changes between the read and write phases of the operation under test.
**Resolution:** §14.3.1 "Extended Test Format" added to 14-conformance.md, documenting `simulate` as an extended input field.
