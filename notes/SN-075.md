**Sections:** §12.1, §6.3, §6.4, §14.1
**Severity:** High (spec architecture)
**Issue:** §12.1 step 6 says: "If the type was specified explicitly, the final record MUST satisfy that type's match rules (§6.3–§6.4)." This step is part of the Create operation, which is a Level 1 requirement. However, evaluating match rules (path_glob, fields_present, where conditions) is a Level 2 capability. This creates a contradiction:
- A Level 1 implementation must implement Create (§12.1)
- §12.1 step 6 requires evaluating match rules
- Match rule evaluation is not required until Level 2

This was partially introduced by SN-066, which added the match_failed check. The resolution there put the tests at Level 1, compounding the issue.
**Proposed resolution:** Add a conformance qualifier to §12.1 step 6: "This check applies when the implementation supports match rule evaluation (Level 2+). Level 1 implementations MAY skip this step." Alternatively, restructure so that Level 1 only checks the explicit type exists and Level 2 adds the match rule satisfaction check.
**Resolution:** §12.1 step 6 now applies only when match rules are supported; Level 1 may skip.
