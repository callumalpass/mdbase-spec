Runtime profile 0.1

Portable automation contracts

The runtime profile defines typed Markdown records for events, actions, and workflows. Applications provide the executable behavior and decide what is authorized.

Read the runtime profile See an example
Model

Declare, provide, execute

The collection stores portable contracts. A runtime connects those contracts to live application behavior.

  1. 01

    Collection declares

    Event, action, and workflow records describe what can happen and how the pieces connect.

  2. 02

    Provider implements

    An application registers the events it can emit and the actions it can perform.

  3. 03

    Host executes

    The host validates contracts and policy, authorizes the action, and dispatches it.

Example

A drop event updates one record

This abridged workflow comes from the canonical Canvas runtime fixture. 

type: workflow
id: canvas.zone.set-status
version: 1

triggers:
  - id: drop-on-status-zone
    event: canvas.drop

steps:
  - id: patch-task-status
    action: mdbase.record.patch
    input:
      path:
        $expr: 'event.payload.file.path'
      patch:
        status:
          $expr: 'event.payload.zone.id'

The event supplies a file path and destination zone.

The workflow maps those values into a record patch action.

The host validates the contracts and active policy before dispatch.
